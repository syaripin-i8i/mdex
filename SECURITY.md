# Security Policy

## Supported Versions

Security fixes are provided for the latest `0.x` line on a best-effort basis.

| Version | Supported |
|---|---|
| 0.1.x | Yes |
| < 0.1.0 | No |

## Reporting a Vulnerability

Do not open public issues for suspected vulnerabilities.

Please use GitHub Security Advisories (private reporting):

1. Open the repository Security tab.
2. Choose "Report a vulnerability".
3. Include reproduction steps, affected command(s), and expected impact.

Direct link: https://github.com/syaripin-i8i/mdex/security/advisories/new

If GitHub private reporting is unavailable, use X (Twitter) DM: `@syaripin_i8i`, and include:

- affected version/commit
- proof-of-concept input
- impact and exploitability notes
- suggested mitigation (optional)

## Response Targets

- Initial acknowledgment: within 3 business days
- Triage decision: within 7 business days
- Fix or mitigation plan: as soon as practical after triage

These are best-effort goals, not guarantees.

## Disclosure Process

- Reports are triaged privately.
- A fix is prepared and validated.
- Public disclosure follows coordinated release timing.
- `CHANGELOG.md` documents security-impacting fixes.
