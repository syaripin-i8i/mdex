## 背景
前タスク `T20260421180146` で `tomllib` 非互換と artifact hash mismatch には対応したが、最新 push CI はまだ失敗している。

対象 commit:
- `5ee94c399406afa1932e5317a9e394f4cd248709`
- message: `Fix hash-locked installs across Python versions and CI matrix`

失敗 run:
- `https://github.com/syaripin-i8i/mdex/actions/runs/24738745515`

確認できた事実:
- `pytest + coverage (ubuntu-latest, py3.12)` の `Install dependencies from lock` で以下が発生
  - `ERROR: In --require-hashes mode, all requirements must have their versions pinned with ==. These do not:`
  - `typing_extensions<5.0,>=4.6 ... (from cyclonedx-python-lib==11.7.0 -> -r tmp requirements)`
- 同じエラーが `ubuntu-latest py3.10` / `macos-latest py3.11` でも再現
- `build smoke` と `pip-audit` も同じ install step で停止

追加観察:
- `C:\Codex\mdex\pylock.toml` には `cyclonedx-python-lib==11.7.0` はあるが `typing-extensions` は存在しない
- `C:\Codex\mdex\.github\scripts\install_from_pylock.py` は `pylock.toml` に載っている package だけを `--require-hashes` 用 requirements に変換している
- そのため clean CI 環境では `cyclonedx-python-lib` の推移依存 `typing_extensions<5.0,>=4.6` が未 pin requirement として解決され、pip が hash mode で拒否する

つまり前回の hash mismatch は解消したが、**lock が fully-closed ではないまま `--require-hashes` を使っている** ことが残課題。

## 目的
`--require-hashes` を維持したまま、推移依存まで閉じた install を実現し、CI matrix を green に戻す。

## 対象
- `C:\Codex\mdex\.github\scripts\install_from_pylock.py`
- `C:\Codex\mdex\pylock.toml`
- `C:\Codex\mdex\.github\locks\pypi_release_hashes.json`
- `C:\Codex\mdex\.github\scripts\export_release_hashes.py`
- `C:\Codex\mdex\tests\test_install_from_pylock_script.py`
- 必要なら lock 更新手順を説明する `README.md` / docs

## 完了条件

3. `build smoke` と `pip-audit` も同じ lock install で通る

   - 推移依存未 pin を検知する回帰テスト
   - lock / supplemental hash 生成手順に「推移依存を fully close する」手順と検証コマンドを明記
6. GitHub Actions の実 run で green を確認できる
