## 目的
enrich を省略したノードが蓄積すると find/related/context の精度がじわじわ落ちるが、どのノードが seed summary のままか見えない。
`mdex stale --db <db>` で summary_source=seed のまま一定期間更新されていないノードを列挙できるようにする。

## 仕様
- コマンド: `mdex stale --db <sqlite> [--days <n>] [--format table|json]`
- --days: 最終更新から何日以上経過したものを対象にするか（デフォルト 30）
- 出力: id / title / type / status / summary_source / updated を含む JSON 配列（デフォルト）
- summary_source=agent のノードは除外する
- store.py に list_stale_nodes(db_path, days) を追加し、cli.py から呼ぶ
