## Description

タスクファイル: `C:\Codex\infra\tasks\pending\{ID}.md`
実装先: `C:\Codex\mdex`

## 背景
0.1.0 では transaction rollback を導入したが、temp DB+`os.replace()` による full atomic replacement ではない。SQLite ファイル破壊の worst case はまだ残る。監査の宿題。

## 目的
`write_sqlite` を「temp DB に書いてから os.replace」方式に切替、任意のタイミングで中断されても既存 DB が無傷。

## 対象ファイル
- `runtime/indexer.py::write_sqlite`
- `tests/` (fault injection regression test を強化)

## 完了条件
- temp DB 生成 → INSERT 完了 → `os.replace(temp_path, final_path)` で原子的入替
- `node_overrides` は新 DB に事前コピー(または migrate)してから replace
- kill -9 相当の強制中断でも既存 DB が無傷(fault injection test)
- 既存 test 全 green

## 非目標
- WAL mode 切替(別 patch)
- SQLite backup API 使用(shutil.move で十分)

## 着手前
- 現行 transaction 実装を Read、どこを temp 化するか設計
