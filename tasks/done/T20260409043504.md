## 目的
`mdex first <node-id>` コマンドを追加する。
指定ノードを理解する前に先に読むべきノード群を順序付きで返す。
`related`（横方向の関連探索）とは異なり、`first` は縦方向の前提列を返す。

## 設計
- `depends_on` エッジを逆辿りして前提ノードを収集する
- 並び順: 前提が先、依存先が後（読み順 = 根に近いものから）
- サイクル検出が必要（循環 depends_on があっても無限ループしない）
- resolved=true のエッジのみ対象

## 対象ファイル
- runtime/resolver.py に `prerequisite_order(node_id, db_path, limit)` を追加
  - depends_on エッジを BFS/DFS で逆辿り（node_id → depends_on先 → さらにその先）
  - トポロジカル順（前提が先）で返す
  - サイクルは訪問済みセットで回避
- runtime/cli.py に `_cmd_first` と subparser `first` を追加
  - 引数: `node`（位置引数）, `--db`, `--limit`（デフォルト10）
  - 出力: JSON（related と同形式）
    ```
    {"node": {...}, "prerequisites": [...]}
    ```
- tests/test_mdex.py に検証を追加
  - A depends_on B depends_on C の連鎖で C → B → A の順に返ることを確認
  - 循環依存でも無限ループしないことを確認

## 完了条件
- `mdex first <node-id> --db <db>` で前提ノードが読み順で返る
- 前提なしの場合は `prerequisites: []`
- サイクルで無限ループしない
- `python -m pytest -q` が通る
