## Description

実装先: `mdex/start.py`, `mdex/context.py`, `README.md`, `schemas/start.schema.json`, `schemas/context.schema.json`, `tests/`

## Background

現状の `recommended_next_actions` は `"open spec/b.md"` のような自然文文字列で、AI には読めるが実行器には扱いづらい。既存 field を壊さず、構造化版を additive に持たせると downstream が安定する。

## Done When

1. `start` と `context` の成功 payload に additive な `recommended_next_actions_v2` を追加する
2. 各 action が `command`, `args`, `reason` を持ち、既存の文字列配列と意味対応している
3. 既存 `recommended_next_actions` は互換維持で残す
4. README / schema / examples / tests が v1 と v2 の併存を説明している

## Non-goals

- 既存 `recommended_next_actions` を削除すること
- すべての自然文説明を構造体だけに置き換えること

## Result

- 実施内容: `mdex/context.py` と `mdex/start.py` に `recommended_next_actions_v2` を追加。各 action を `command` / `args` / `reason` の構造体で返すようにし、既存 `recommended_next_actions`（文字列配列）は互換維持。`schemas/start.schema.json` と `schemas/context.schema.json`、`README.md`、関連テストを更新。
- 確認方法: `python -m pytest -q tests/test_context.py tests/test_start.py tests/test_phase_a_cli.py::test_context_actionable_and_start_contract` と `python -m pytest -q` で通過確認。
- 備考: v1/v2 併存で downstream 移行しやすい構成。
