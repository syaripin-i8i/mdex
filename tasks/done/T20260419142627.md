## 目的

`mdex finish --dry-run` の「成功時に何が出力されれば OK か」を AI が初見で誤読しないよう、出力形状の意味と判定基準を明示する。

## 背景

実運用フィードバック: Claude が初回で `finish --dry-run` を実行したとき、`changed_files: []`, `enrich_candidates: []`, `requires_manual_targeting: false` という空配列中心の出力を見て「失敗？」と一瞬思った。実際は「変更対象がなく正常終了」の意味だが、**空配列 = 正常** の慣習が README で明示されていない。

## 完了条件

1. README の Output Contract 節に finish の成功パターン説明を追加:
   - 空配列 = 「該当なし・正常終了」の意味
   - `requires_manual_targeting: true` が出た場合の次アクション
   - `dry_run: true` と実行モードの違い
2. schemas/finish.schema.json の各フィールドに description を拡充（既存があれば補強）
3. README に「成功と失敗の判別ルール」を 1 行追加:
   - 成功 = stdout JSON 返却（内容が空でも成功）
   - 失敗 = stderr JSON + exit != 0
4. 既存の再現サンプル節の finish 出力例に「空配列は正常終了の意味」というコメントを 1 行添える

## 非ゴール

- finish の動作仕様変更
- エラーメッセージの刷新
