## 目的
`mdex context "<query>" --db <db>` コマンドを追加する。
Claude/Codex がタスク着手前に呼び出し、読むべきノードを優先順位つきで受け取るための入口。
AI 非依存・高速・決定論的。クエリのたびに API 呼び出しは発生しない。

## 設計方針
- クエリ文字列からキーワードを抽出（単純分割、AI なし）
- store.search_nodes() で title/summary/tags マッチ候補を収集
- グラフ近傍（related edges）で候補を補完
- スコアリング要素: キーワードマッチ強度・グラフ近接度・type（設計メモ・AGENT.md を優遇）・status・更新日時
- --budget はガイド。大幅超過を避ける程度のソフト上限。重要ノードは超過しても返す

## 対象ファイル
- runtime/tokens.py を新規作成
  - ノード本文の文字数からトークン数を見積もる関数（文字数÷4 程度のラフ推定、tiktoken があれば使う）
- runtime/context.py を新規作成
  - select_context(query, db_path, budget, limit) を実装
  - 上記スコアリングロジックを閉じる
- store.py に list_edges_for_nodes(db_path, node_ids) を追加（グラフ近傍補完用）
- cli.py に _cmd_context と subparser context を追加
  - 引数: query（位置引数）, --db, --budget（デフォルト 4000）, --limit（デフォルト 10）
  - 出力: JSON { query, nodes: [{id, priority, reason, estimated_tokens}], total_tokens, budget }

## 完了条件
- mdex context "..." --db <db> でノード一覧が JSON で返る
- budget を大幅に超える場合は絞られる（ソフト上限）
- python -m pytest -q が通る
