## ノードスキーマ（AGENT.md に記載）

```json
{
  "id": "relative/path/to/file.md",
  "title": "...",
  "type": "decision|task|design|log|spec|reference|unknown",
  "project": "...",
  "status": "active|done|draft|archived|unknown",
  "summary": "...",
  "tags": [],
  "updated": "ISO date string",
  "links_to": ["other/file.md"],
  "depends_on": ["other/file.md"],
  "relates_to": ["other/file.md"]
}
```

## type 判定ロジック

1. frontmatter の `type` フィールドがあればそれを使う
2. なければファイルパスのディレクトリ名で `control/scan_config.json` の `node_type_map` と照合
3. どちらもマッチしなければ `unknown`

## エッジ生成ロジック

- `[[wikilink]]` / `[text](file.md)` → `links_to` エッジ
- frontmatter の `depends_on` リスト → `depends_on` エッジ
- frontmatter の `relates_to` リスト → `relates_to` エッジ
- `[[link]]` の解決: scan root からの相対パスで `.md` を補完してマッチを試みる（ベストエフォート）

## 出力形式

```json
{
  "generated": "ISO timestamp",
  "scan_root": "...",
  "nodes": [...],
  "edges": [{"from": "...", "to": "...", "type": "..."}]
}
```
