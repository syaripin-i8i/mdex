## 背景
現状の `install_from_pylock.py` は `name==version` の requirements を組み立てて pip install しているだけで、artifact hash 検証をしていない。監査で「供給網の厳密性が一段甘い」と指摘された宿題。

## 目的
同じ commit から同じ artifact が install されることを hash レベルで保証する。

## 対象ファイル
- `install_from_pylock.py` (または pip 25.2+ の `pip install --lock pylock.toml` 相当に置換)
- `.github/workflows/ci.yml` (lock install step)

## 完了条件
- `pylock.toml` の sha256 フィールドを使った `pip install --require-hashes` 相当の install
- または pip native の `pip install --lock` を使用(pip 25.2+)
- CI でこの install を通す
- `pip-audit` 結果が変わらないこと

## 非目標
- lockfile 自動生成ツールの切替(uv / pip-tools 等)

## 着手前
