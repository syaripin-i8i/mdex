# Task: 永続化修正: node_overrides テーブルで agent summary を scan 越しに保護

## Description

タスクファイル: C:\Codex\infra\tasks\pending\
実装先: C:\Codex\mdex

## 問題
write_sqlite() が毎回 DROP TABLE nodes / edges / index_metadata を実行するため、
enrich で積み上げた agent summary が次回 scan で全消えする。
「書けるが保存されない記憶」になっており、思想と実装が矛盾している。

## 解決方針（案B）
seed index（scan 由来）と agent write-back（enrich 由来）を別テーブルに分離する。

## 変更内容

### 1. node_overrides テーブルを新設
スキーマ:
- id TEXT PRIMARY KEY
- summary TEXT NOT NULL
- summary_source TEXT NOT NULL（常に "agent"）
- summary_updated TEXT NOT NULL（ISO 8601）

### 2. indexer.py の write_sqlite() を修正
- DROP TABLE の対象から node_overrides を外す
- node_overrides は CREATE TABLE IF NOT EXISTS で初回のみ作成
- nodes / edges / index_metadata は従来通り DROP して再作成

### 3. store.py を修正
- get_node() : nodes を取得後、node_overrides に同 id があれば summary / summary_source / summary_updated を上書きして返す
- list_nodes() : 同様に override を適用する。Python 側でのマージでよい（JOIN 不要）
- update_node_summary() : nodes ではなく node_overrides に UPSERT する

### 4. enrich.py を修正
- update_node_summary() 経由で node_overrides に書くだけ（store.py 側が対応すれば自動的に正しくなる）

### 5. テスト追加
- enrich → scan → get_node() でも agent summary が保持されることを確認
- scan 後も summary_source が "agent" のまま返ることを確認
- enrich していない node は summary_source が "seed" であることを確認

## 完了条件
- enrich → scan のサイクルで agent summary が消えない
- get_node() / list_nodes() が node_overrides を優先して返す
- node_overrides は DROP されない
- python -m pytest -q が通る
