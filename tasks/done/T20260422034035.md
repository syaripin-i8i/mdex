## 背景
commit `a331728b4f0126a70c79bc565638204849bc0d68` の CI run `24739848708` では、install bootstrap は解消した一方で `pytest + coverage` の `py3.10` 3 job が失敗した。

対象 run:
- `https://github.com/syaripin-i8i/mdex/actions/runs/24739848708`

確認できた事実:
- 失敗は `ubuntu-latest, py3.10` / `macos-latest, py3.10` / `windows-latest, py3.10`
- 3 job とも `Install dependencies from lock` は成功
- 失敗箇所は `Run tests with coverage gate`
- 共通エラー:
  - `ImportError while importing test module tests/test_install_from_pylock_script.py`
  - `tests/test_install_from_pylock_script.py:6: in <module>`
  - `import tomllib`
  - `ModuleNotFoundError: No module named 'tomllib'`

つまり、アプリ本体ではなく **test file が Python 3.10 非対応 import を持っている** のが残件。

## 目的
`tests/test_install_from_pylock_script.py` を Python 3.10 でも collect できるようにし、9-cell pytest matrix を green に戻す。

## 対象
- `C:\Codex\mdex\tests\test_install_from_pylock_script.py`

## 完了条件
1. `py3.10` で `tests/test_install_from_pylock_script.py` が collection error を起こさない
2. `python -m pytest -q tests/test_install_from_pylock_script.py` が通る
3. 可能なら `tomllib` 不在時の fallback import をローカルで確認する
4. GitHub Actions の再 run green を確認できたら `done/` に移動する

## Result

- 実施内容: `tests/test_install_from_pylock_script.py` の `tomllib` 直 import 問題を修正し、pytest 側では `tests/conftest.py` で `py3.10` 用に `tomli -> tomllib` shim を先に注入するよう変更。加えて bootstrap scripts の `tomllib` import が `ModuleNotFoundError` guard 付きであることを検査する回帰テストを追加し、`CONTRIBUTING.md` と checklist に Python 3.10 互換 import の注意点を追記。
- 確認方法: `python -m pytest -q tests/test_install_from_pylock_script.py`、`python -m pytest -q`、および GitHub Actions CI run `24740055135` (`https://github.com/syaripin-i8i/mdex/actions/runs/24740055135`) の green を確認。
- 備考: `.gitignore` には `.pytest_cache/`、`.coverage`、`coverage.xml`、`htmlcov/` を追加して、今回のような test/debug 作業で出る生成物の混入も避けるようにした。
