# Task: [mdex/0.1.1] CLI JSON 出力を `_emit_payload()` 経由へ統一する

**ID**: T20260424084815
**Project**: mdex
**Priority**: high
**Status**: done
**Created**: 2026-04-24T08:48:15+09:00

## Description

実装先: `mdex/cli.py`, 必要なら `tests/`

## Background

README は「成功は `stdout` JSON、失敗は `stderr` JSON」の契約を明示している。現状は `_emit_payload()` に compact JSON の経路がある一方、`scan` / `context` / `start` / `finish` などで `json.dumps(..., indent=2)` を直接呼ぶ箇所が残っており、整形ポリシーが分散している。

## Done When

1. `_emit_payload()` が `stderr` と pretty/compact の両方を統一的に扱える
2. JSON を返す全コマンドが `_emit_payload()` 経由になり、直接 `json.dumps()` する経路が残らない
3. 成功/失敗の stream 契約は維持しつつ、payload field 名と意味は変わらない
4. CLI テストで compact/pretty の両方が崩れないことを確認する

## Non-goals

- JSON field の rename や schema 変更
- table 出力の仕様変更

## Result

- 実施内容: `mdex/cli.py` の `_emit_payload()` を `stderr` + pretty/compact 両対応へ拡張し、JSON を返す全コマンド出力（`scan/context/start/finish/...`）を `_emit_payload()` 経由へ統一。`print(json.dumps(...))` の直呼び経路を除去。
- 確認方法: `tests/test_cli_payload.py` を追加して pretty/compact/stderr の挙動を検証し、`python -m pytest -q` で全体通過を確認。
- 備考: 成功系は pretty JSON、失敗系は compact JSON を維持。
