## 背景
mdex は "protocol-first CLI" を掲げているが、現状 JSON 出力契約は README / docs/design.md の prose のみで機械可読仕様が無い。agent-first を標榜する以上、schema をリポジトリに固定し、破壊的変更ポリシーを明文化する必要がある。

## 対象
- schemas/ ディレクトリ新設（scan.schema.json / start.schema.json / context.schema.json / impact.schema.json / finish.schema.json）
- 各 schema は JSON Schema draft 2020-12 準拠
- `$id` / `$schema` / `title` / `description` / `required` / `properties` を完備
- docs/schema_versioning.md 新規作成（semver + deprecation ポリシー + 破壊的変更の告知手順）
- README にも schema 所在と版方針を記載
- tests/ に「各 CLI コマンドの実出力が schema を満たす」ことを検証する test を追加（jsonschema ライブラリ利用可。必要なら pyproject の dev 依存に追加）

## 完了条件
- schemas/*.schema.json が揃い、各コマンドの現行出力に完全整合
- contract test が全 commands を網羅し green

- README に schemas/ への導線追加
- pytest green

## 非目標
