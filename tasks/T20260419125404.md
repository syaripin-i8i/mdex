## 目的
`write_sqlite` が既存 DB 上で DROP → CREATE → INSERT している現状を、**明示 transaction で all-or-nothing** に包む。中断・障害時に既存の良い DB を壊さない。

## 対象ファイル
- `runtime/indexer.py::write_sqlite`
- `tests/` (fault injection regression test)

## 完了条件
- DROP/CREATE/INSERT 一連を `conn.execute("BEGIN")` / `conn.commit()` / 例外時 `conn.rollback()` で包む
- `node_overrides` テーブルは transaction 内で SELECT → 保持 → 再 INSERT(既存ロジック維持)
- fault injection test: `write_sqlite` の途中で例外を monkeypatch で注入し、既存 DB が(a) rollback で無傷、(b) 少なくとも `node_overrides` が失われないこと を assert
- 既存の scan 系 test が通る
- `pytest -q` all green

## 非目標

- WAL mode 切替
