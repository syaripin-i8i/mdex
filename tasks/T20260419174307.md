## 背景
2026-04-19 に mdex-cli 0.1.0 を editable install (`pip install -e .`) したところ、site-packages に top-level `runtime/` パッケージが露出し、yura 側の namespace package `C:\Codex\yura\runtime\` をシャドウして ops-once を全停止させた（yura 側 T20260419170250 で対処済み、bat の PYTHONPATH + `-m` + `__init__.py` で回避）。今回は yura 側で防御した状態だが、mdex-cli が top-level `runtime` を握り続けるのは他プロジェクト・将来インストール環境でも事故が再発する。

## 原因の設定
`C:\Codex\mdex\pyproject.toml`:
```toml
[project.scripts]
mdex = "runtime.cli:main"

[tool.setuptools]
packages = ["runtime"]
```
で `C:\Codex\mdex\runtime\` がそのまま top-level package として公開されている。editable install の finder (`__editable___mdex_0_1_0_finder.py`) が `import runtime` を mdex 配下に引っ張っている。
