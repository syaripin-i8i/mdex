## Description

タスクファイル: C:\Codex\infra\tasks\pending\{ID}.md

## 背景
docs/support_matrix.md は Linux/macOS/Windows を謳うが CI は ubuntu のみ。オーナーは3OS 全てで実利用する予定なので matrix を拡張する。ただし maintenance 体制は solo なので、サポート表現は「primary: Linux / best-effort: macOS, Windows」と現実的なトーンにする。

## 対象
- .github/workflows/ci.yml: `test` job の matrix に `os: [ubuntu-latest, macos-latest, windows-latest]` を追加、`runs-on: ${{ matrix.os }}`、`fail-fast: false`。Python は 3.10/3.11/3.12 継続
- docs/support_matrix.md: サポート宣言のトーンを「primary (maintained & CI-tested): ubuntu-latest / best-effort (CI-tested, community issues welcome): macos-latest, windows-latest」のように整理。応答 SLA も現実的な range に
- README.md: CI バッジが matrix 全体を表すか、少なくとも primary を表示するよう調整
- 既存テストが OS 依存（改行・path separator・subprocess・tempfile permission 等）で落ちる場合は、原因を特定して修正。修正コストが大きく妥当性が薄いものは `@pytest.mark.skipif(sys.platform == "win32", ...)` 等で明示スキップし、スキップ理由と follow-up 課題を docstring に残す

## 完了条件
1. CI が 3 OS × 3 Python = 9 job 構成で動作
2. 少なくとも ubuntu は 100% green
3. macOS / Windows は green を目指し、落ちたテストはスキップ理由と follow-up が追跡可能な状態で残す（"全部スキップ" は不可）
4. support_matrix.md のトーンが primary / best-effort に分離されている
5. 既存 contract test / readme examples test が全 OS で green（スキップ無しで通せるのが理想）
6. pytest green（ローカルでも）

## 非目標
- self-hosted runner / custom image
- Python 3.13 以降対応
- wheel build / PyPI publish（release workflow 別タスク）

## 着手前
- Windows で既知の subprocess ハンドル問題（`[WinError 6]`）が pytest から git を呼ぶ path で起きる場合がある。`tests/test_phase_a_cli.py::_init_git_repo` 等の subprocess 呼び出しを再確認し、GitHub Actions の Windows runner と Claude Code の bash shell で挙動が違う可能性を考慮する（GitHub Actions 側は別環境なので再現しない可能性が高い）
- macOS 固有: case-insensitive FS、BSD 系 coreutils の差分

## Result

CI matrix を 3 OS × 3 Python = 9 job に拡張、support_matrix を primary(ubuntu) / best-effort(macos, windows) に整理。ubuntu/macOS は全通過、Windows 3 job は UnicodeEncodeError で失敗中（別途対応）。
