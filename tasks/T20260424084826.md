# Task: [mdex/0.1.1] `start` に index freshness と入口診断 field を追加する

**ID**: T20260424084826
**Project**: mdex
**Priority**: high
**Status**: pending
**Created**: 2026-04-24T08:48:26+09:00

## Description

実装先: `mdex/start.py`, `mdex/context.py`, `README.md`, `schemas/start.schema.json`, `tests/`

## Background

現状の `start` は実質 `context --actionable` の薄いラッパーで、README が期待する「作業開始時の入口決定コマンド」としての固有価値が弱い。特に index の鮮度不足、low confidence、候補空振り時の理由が machine-readable に返ってこない。

## Done When

1. `start` の成功 payload に additive な `entrypoint_reason` を追加し、「なぜその入口なのか」を固定 field で返す
2. `index_status` を返し、index が古いと判断した場合は `recommended_next_actions` に `run mdex scan` を含める
3. `confidence < 0.6` や `recommended_read_order == []` のとき、原因を区別できる reason field を返す
4. README / schema / examples / tests が新 field と判定基準を説明している

## Non-goals

- `context` 本体の ranking アルゴリズムを大きく作り替えること
- code search や全文検索を `start` に追加すること

## Result

- 実施内容: `mdex/start.py` に `index_status` の鮮度診断（`fresh/stale/age_hours/stale_after_hours/reason`）を追加し、`entrypoint_reason` を新設。`confidence < 0.6` / `recommended_read_order == []` / index stale を区別可能な reason 値で返却。stale 判定時は `recommended_next_actions` に `run mdex scan` を付与。
- 確認方法: `tests/test_start.py` と `tests/test_phase_a_cli.py::test_context_actionable_and_start_contract` を更新し、`python -m pytest -q` で通過確認。
- 備考: `start` の field は additive で既存 payload 意味を維持。
