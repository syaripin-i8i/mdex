## Description

## 目的

`C:\Codex\mdex\README.md` の enrich・context 周辺の記述を、現在の運用実態に合わせて更新する。

## 背景

現在の README には以下の齟齬がある：
1. `enrich` と `context` が "experimental / Phase 3 / 評価中" 扱いのまま → 実際は主要ワークフローとして稼働中
2. `--db` を省略すると `mdex_index.db`（ローカル小DB）に書かれてしまうが、その注意書きがない
3. サマリの書き方ガイドがなく、品質にばらつきが出る
4. `context` が「補助コマンド（複合選別が必要なときだけ使う）」と書かれているが、実際はタスク着手前の**最初のステップ**

## 対象ファイル

`C:\Codex\mdex\README.md`

## 変更内容

### 1. コマンド責務テーブルの修正（「コマンド責務（境界）」セクション）

`context` と `enrich` の「用途」列から experimental の意味合いを除き、実態に合わせる：

| コマンド | 用途（修正後） |
|---------|--------------|
| `context` | タスク着手前に読むべき .md を絞り込む主要入口。まずこれを呼ぶ |
| `enrich`  | ドキュメントを読んだあと summary を更新する write-back。索引品質を上げる |

### 2. 「エージェントの使い方」セクションの構成変更

現在の構成：
- タスク着手時の本流（Phase 2）
- 補助コマンド（experimental / Phase 3）
- ファイルを読んだ後（write-back）

修正後の構成：
- タスク着手時の標準フロー（context → find/first/related の順に説明）
- ファイルを読んだ後（enrich）

`context` を「補助コマンド」から「標準フローの最初のステップ」に格上げする。
"experimental" / "Phase 3" / "評価中" の注記を削除する。

### 3. enrich セクションに以下を追記

**`--db` は必ず明示すること**

```bash
# NG: --db を省略すると mdex_index.db（ローカルDB）に書かれる
mdex enrich foo.md --summary "..."

# OK: 常に対象DBを明示する
mdex enrich foo.md --db C:\Codex\mdex\mdex_codex_index.db --summary "..."
```

**サマリの書き方**

「別のエージェントがこのファイルを読むべきか判断するための情報」を書く。
- 何が書いてあるか（内容の要点）
- どんな制約・判断・ルールを含むか
- いつ参照すべきか

```
悪い例: 「このファイルはキャラクター設定を含む。」
良い例: 「ゆらの口調・禁止表現・ペルソナを定義。発話品質に関わる作業の前に参照する。」
```

**既存サマリの上書き**

```bash
# agent summary が既に存在する場合は --force を付ける
mdex enrich foo.md --db ... --summary "..." --force
```

### 4. 「作業サイクル」セクションの冒頭に一文追加

「索引を作っただけでは半分です。」の前に：

> エージェントは作業前に `mdex context` でドキュメントを絞り、読んだあとに `mdex enrich` でサマリを更新する。このサイクルを回すほど索引の精度が上がる。

### 5. README 冒頭の注記は変えない

「このツールは AI エージェントのために作られています」の冒頭文・コンセプト説明（「何を解くツールか」「なぜ AI の中だけで完結させてはいけないか」）は変更しない。
