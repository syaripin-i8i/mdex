## 背景
commit `b641738ee26c763488fcb8996437952edd97049c` (`Harden CI with hash-locked installs and add Semgrep checks`) 後、push CI が失敗した。

失敗 run:
- `https://github.com/syaripin-i8i/mdex/actions/runs/24737199823`

確認できた事実:
- `pytest + coverage (ubuntu-latest, py3.10)` は `Install dependencies from lock` で `ModuleNotFoundError: No module named 'tomllib'`
- `pytest + coverage (macos-latest, py3.11)` は `charset-normalizer==3.4.7` の hash mismatch (`Expected sha256 92a0...`, `Got 7641...`)
- `pytest + coverage (ubuntu-latest, py3.12)` は同じく `charset-normalizer==3.4.7` の hash mismatch (`Got 5649...`)
- `pytest + coverage (windows-latest, py3.12)` も同じく hash mismatch (`Got 5ed6...`)
- `build smoke` と `pip-audit` も同じ install step で停止

追加観察:
- `C:\Codex\mdex\.github\scripts\install_from_pylock.py` は `import tomllib` を直書きしており、Python 3.10 では import 時点で落ちる
- `C:\Codex\mdex\pylock.toml` の `charset-normalizer` エントリは `charset_normalizer-3.4.7-cp314-cp314-win_amd64.whl` 1本とその hash しか持っていない
- 同様に `tomli`, `msgpack`, `pyyaml`, `rpds-py` なども `cp314-win_amd64` wheel 起点の lock になっており、3 OS × 3 Python matrix に対して `--require-hashes` でそのまま使える形ではない

つまり今回の CI 赤は 2 系統:
1. `install_from_pylock.py` の Python 3.10 非互換 (`tomllib` fallback 不足)
2. `pylock.toml` / install strategy が matrix 非対応のまま hash 検証を厳格化したため、3.11/3.12 系で artifact hash mismatch

## 目的
hash 検証を維持したまま、support matrix (`ubuntu/macos/windows × 3.10/3.11/3.12`) で CI を green に戻す。

## 対象
- `C:\Codex\mdex\.github\scripts\install_from_pylock.py`
- `C:\Codex\mdex\pylock.toml`
- `C:\Codex\mdex\.github\workflows\ci.yml`
- 必要なら `C:\Codex\mdex\tests\test_install_from_pylock_script.py`
- 必要なら lock 生成手順を説明する doc / README

## 完了条件
1. Python 3.10 job で `install_from_pylock.py` が import 時点で落ちない
2. lock install が 3 OS × 3 Python の CI matrix で通る
3. `build smoke` と `pip-audit` も同じ lock install で通る
4. hash 検証は維持する（`--require-hashes` 廃止や単なる pin 緩和で逃げない）
5. 再発防止として、少なくとも以下のどちらかが入っている
   - matrix/複数 artifact を前提にした lock 解釈 or lock 再生成の回帰テスト
   - README / docs に「この lock はどの Python/OS を対象に生成するか」の手順固定
6. GitHub Actions の同等 run で green を確認できる

## 非目標
