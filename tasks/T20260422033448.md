## Description

タスクファイル: `C:\Codex\infra\tasks\pending\T20260422033448.md`
実装先: `C:\Codex\mdex`

## 背景
前タスク `T20260421181523` で `--require-hashes` 用の推移依存 closure は入ったが、その変更で lock installer 自体の bootstrap が壊れた。

対象 commit:
- `6e02f28bbfc0c4806e3032593f4eb0c9183387ca`
- message: `Close transitive hash-locked dependencies for require-hashes`

失敗 run:
- `https://github.com/syaripin-i8i/mdex/actions/runs/24739514836`

確認できた事実:
- `pytest + coverage (ubuntu-latest, py3.12)` / `macos-latest, py3.11` / `windows-latest, py3.11` などで `Install dependencies from lock` が即失敗
- 代表ログ:
  - `Traceback (most recent call last):`
  - `File ".../.github/scripts/install_from_pylock.py", line 12, in <module>`
  - `from packaging.markers import default_environment`
  - `ModuleNotFoundError: No module named 'packaging'`
- `windows-latest, py3.12` だけは runner に `packaging 26.0` がプリインストールされていて通過している

つまり、**依存をインストールする前に動く bootstrap script が、先に `packaging` を要求してしまっている** のが今回の回帰。

## 目的
`install_from_pylock.py` が clean CI runner 上でも追加事前依存なしで起動し、`--require-hashes` 用の requirements 生成まで進めるようにする。

## 対象
- `C:\Codex\mdex\.github\scripts\install_from_pylock.py`
- `C:\Codex\mdex\tests\test_install_from_pylock_script.py`
- 必要なら関連ドキュメント / 補助スクリプト

## 完了条件
1. `install_from_pylock.py` が external `packaging` 未導入環境でも import 時に落ちない
2. bootstrap 回帰を捕まえる回帰テストがある
3. `python -m pytest -q tests/test_install_from_pylock_script.py` が通る
4. 可能な範囲で関連ローカル検証を実施し、結果を記録する
5. GitHub Actions の再 run 結果を確認できたら `done/` へ移動する

## 非目標
- `--require-hashes` を外すこと
- closure ロジックそのものの再設計
- Semgrep / CodeQL / gitleaks の別問題修正
