# Task: Normalize completed task frontmatter records

Normalize completed task frontmatter records so mdex scan sees the real task body instead of stale embedded YAML.

## Background

Some completed task records had a `done` frontmatter block prepended while the original `pending` frontmatter remained at the start of the body. That made the first body paragraph look like YAML and kept inline status text stale.

## Result

- Removed duplicate embedded frontmatter blocks from the affected completed task records.
- Preserved the removed `relates_to` values in canonical frontmatter so mdex task graph edges remain intact.
- Updated their inline `**Status**` fields from `pending` to `done`.
- Added a docs consistency regression test that fails when a root task file keeps a second frontmatter block immediately after the real frontmatter.
