# Task: [mdex/contract] `finish --dry-run` の noop 成功を明示する

**ID**: T20260424084943
**Project**: mdex
**Priority**: low
**Status**: pending
**Created**: 2026-04-24T08:49:43+09:00

## Description

実装先: `mdex/finish.py`, `README.md`, `schemas/finish.schema.json`, `tests/`

## Background

`finish --dry-run` は `changed_files: []` / `enrich_candidates: []` でも正常終了だが、空配列を失敗と誤判定する実行器がある。成功であり noop であることを専用 field で返すと downstream が誤読しにくくなる。

## Done When

1. `finish --dry-run` 成功時に additive な `status`, `noop`, `noop_reason` を返せる
2. 何も変更候補がない正常系で `noop: true` になる
3. 変更候補がある正常系では `noop: false` または省略方針が明文化されている
4. README / schema / tests が「空配列でも成功」の契約と新 field の意味を説明している

## Non-goals

- `finish` の主要判定ロジックを変えること
- `changed_files` / `enrich_candidates` を廃止すること

## Result

- 実施内容: `mdex/finish.py` の成功 payload に `status`, `noop`, `noop_reason` を additive 追加。`finish --dry-run` で `changed_files == []` かつ `enrich_candidates == []` の正常系を `noop: true` と明示。候補がある通常系は `noop: false` を返却。`schemas/finish.schema.json`、`README.md`、`tests/test_finish.py` / `tests/test_phase_a_cli.py` / `tests/test_readme_examples.py` を更新。
- 確認方法: `python -m pytest -q tests/test_finish.py tests/test_phase_a_cli.py::test_finish_dry_run_uses_git_changed_files_without_writes tests/test_readme_examples.py` と `python -m pytest -q` で通過確認。
- 備考: 既存 `changed_files` / `enrich_candidates` 契約はそのまま維持。
