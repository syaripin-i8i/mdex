# Task: [mdex/release] release workflow の install 戦略を CI と整合させる

**ID**: T20260424084859
**Project**: mdex
**Priority**: normal
**Status**: pending
**Created**: 2026-04-24T08:48:59+09:00

## Description

実装先: `.github/workflows/release.yml`, `.github/scripts/install_from_pylock.py`, `docs/release_process.md`

## Background

CI は lockfile-driven install を採用しているが、release workflow は `pip install --upgrade pip build twine` で別経路になっている。再現性を優先して統一するか、公開ツールだけは最新版を取る合理性を文書化するか、どちらかに揃える必要がある。

## Done When

1. release workflow の install 方針が 1 つに決まり、実装と `docs/release_process.md` が一致している
2. lockfile install を採る場合は publish に必要な bootstrap 手順まで含めて成立する
3. 最新 `build/twine` を採る場合は、その理由と再現性トレードオフが文書化されている

## Non-goals

- release pipeline 全体を作り直すこと
- CI と release のすべての job を同一化すること

## Result

- 実施内容: `.github/workflows/release.yml` の install step を `install_from_pylock.py` ベースへ変更し、CI と同じ lockfile-driven install 戦略へ統一。`docs/release_process.md` の pre-release/checklist と workflow 手順を同方針へ更新。
- 確認方法: workflow 定義差分を確認し、`python -m pytest -q` および `python -m build` の通過で publish 前提の依存経路が崩れていないことを確認。
- 備考: 最新 `build/twine` 直取りではなく再現性優先の方針に寄せた。
